Visual Studio Code: A Technical Documentation

Introduction

Visual Studio Code (also referred to as VS Code, logo pictured) is a code editor and integrated development environment (IDE) made by Microsoft. It runs on Windows, MacOS,and Linux. It has support for debugging and code previews, multiple languages through extensions, and extensive customisation options. It is the most popular IDE according to the 2025 Stack Overflow Developer Survey.

Features

  • Multilanguage Support - Out of the box, VS Code includes support for common coding languages such as HTML, CSS, Javascript, and support for more languages can be installed through community-made Extensions.

  • Customisability - VS Code can be extended through user-made Extensions, which add features like additional language support, utilities, debuggers, et cetera. From the Extensions page one can also install Themes that change the look of the program.

  • Source Control - VS Code can be integrated with version control systems such as Git through the built-in Source Control menu, allowing users to manage, pull from, and push to repositories directly from the program.

  • Directory-based File Management - VS Code manages files by making use of directories, allowing it to work with any kind of workflow. Users can open one or more directories and use Workspaces to save their configurations.

Installation

VS Code can be installed a number of ways:

  • For Windows, VS Code can be installed using either a graphical installer or through Winget with the command winget install --id Microsoft.VisualStudioCode.

  • For MacOS, users can download the archive, extract it, and move the resulting Visual Studio Code.app to the Applications folder.

  • For Debian- and Ubuntu-based systems, VS Code ships as an installable .deb file that can be installed either through the graphical installation manager or through the command line with sudo dpkg -i [name-of-file].deb.

Usage

Users are presented with a blank screen upon opening VS Code. From there, they can either sign in with a GitHub or Microsoft account to synchronise settings, or start from scratch on the local machine. Users can press Ctrl+O to open a file or Ctrl+K Ctrl+O to open a folder.
